# `haetae`

`haetae` is a battery-included package for Haetae.<br/>
It's intended to be used alone. Every `@haetae/*{:ts}` packages are already preinstalled in it.<br/>
**This package is recommended for the majority of users.**

<Callout>
  **Are you developing a library (e.g. plugin) for Haetae?** <br />
  You can use [`@haetae/core{:ts}`](./core), [`@haetae/utils{:ts}`](./utils),
  [`@haetae/javascript{:ts}`](./javascript), [`@haetae/git{:ts}`](./git), [`@haetae/cli{:ts}`](./cli)
  independently, if you depend on just partial features of some of them.
</Callout>

## `dependencies`

Unlike other `@haetae/*{:ts}` packages, `haetae{:ts}` does not have `peerDependencies`.

**Note**: This might not be exhaustive and lists only Haetae's packages.

- [`@haetae/core{:ts}`](./core)
- [`@haetae/utils{:ts}`](./utils)
- [`@haetae/javascript{:ts}`](./javascript)
- [`@haetae/git{:ts}`](./git)

## Usage

The package `haetae{:ts}` exports other `@haetae/*{:ts}` packages all at once. <br/>
Its source code is _roughly_ like the snippet below.

<TokenLinkCode tokens={{
  "'@haetae/core'": './core',
  "'@haetae/utils'": './utils',
  "'@haetae/javascript'": './javascript',
  "'@haetae/git'": './git',
  "'@haetae/cli'": './cli',
  }}>
```js copy
export * as core from '@haetae/core'
export * as utils from '@haetae/utils'
export * as js from '@haetae/javascript'
export * as git from '@haetae/git'
export * as cli from '@haetae/cli'
export const pkg = { ... }
```
</TokenLinkCode>

So you probably use `haetae` on the config file like this.

<TokenLinkCode tokens={{'configure': './haetae#configure'}}>
```js copy filename="haetae.config.js"
import { configure, utils, js, git, cli, pkg } from 'haetae'

export default configure({
  // Other options are omitted for brevity.
  commands: {
    // ...
  },
})
```
</TokenLinkCode>

## CLI

`haetae{:ts}` is CLI as well as API.<br/>
`haetae{:ts}` provides exactly same CLI features as [`@haetae/cli{:ts}`](./cli).

**For detailed usage and description as CLI, refer to [CLI](../cli).**

## API

### `pkg`

Refer to [introduction#pkg](./introduction#pkg).

### <Link id="dollar" href="#dollar">`$`</Link>

This is identical to [`utils.${:ts}`](./utils#dollar).

### `configure`

This is identical to [`core.configure{:ts}`](./core#configure).

### `core`

Refer to [`@haetae/core{:ts}`](./core).

### `utils`

Refer to [`@haetae/utils{:ts}`](./utils).

### `js`

Refer to [`@haetae/javascript{:ts}`](./javascript).

### `git`

Refer to [`@haetae/git{:ts}`](./git).

### `cli`

The exported `cli{:ts}` is API of [`@haetae/cli{:ts}`](./cli).<br/>
You probably don't need it.<br/>
Because, in general, the CLI features are used as CLI, not as API.<br/>
If you really need the API, refer to [`@haetae/cli{:ts}`](./cli).<br/>
